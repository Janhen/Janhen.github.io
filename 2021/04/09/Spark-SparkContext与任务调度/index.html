Spark Context

SparkContext 涉及网络通信、分布式、消息、存储、计算、调度、缓存、度量、清理、文件服务和 UI 等方面，是程序的入口点。

SparkContext 内部组件

  • sparkConf： 设置 K-V 格式的参数，配置的优先级高于属性文件
  • sparkEnv： Spark 的运行环境, 所有的 Executor 都有对应的指定环境，开发者 API，无法对外调用
    • RpcEnv: 用于组件之间的通信
    • ShuffleManager： 管理本地和远程的 Block 数据的 Shuffle 操作，根据 spark.shuffle.manager 进行指定，默认使用的是 SortShuffleManager
    • BlockManager： 缓存持久化管理，为 Spark 的 Storage 模块，本身是 Master / Slave 的
    • MemoryManager： 内存管理, 默认使用同一内存管理 UnifiedMemoryManager
    • SerializerManager： 序列化管理器
    • serializer： 序列化器
    • MapOutPutTracker: 跟踪 Map 阶段任务的输出状态，方便 reduce 阶段任务获取地址和中间结果。
    • BroadcaseManager： 广播变量管理，使用 BT 协议分发广播变量
    • MetricsSystem: 度量系统
    • SecurityManager：安全管理
  • ConsoleProgressBar: _progressBar
  • SparkUI:
  • EventLoggingListener:
  • ExecutorAllocationManager:
  • ContextCleaner:
  • MetricsSystem： 指标管理
  • 任务调度相关的：
    • DAGScheduler： DAG 调度器, 负责创建 job，将 DAG 的 RDD 划分为不同的 Stage，提交 Stage
    • TaskScheduler： 任务调度器，按照调度算法对 ClusterManager 已经分配给应用程序的资源进行二次调度后分配任务，调度的 Task 是 DAGScheduler 创建的，DAGScheduler 是 TaskScheduler 的前置调度器。
    • SchedulerBackend： 用于对接不同的资源管理系统

方法：

  • runJob: action 内部调用的
  • submitJob:
  • submitMapStage: 内部 API
  • cancelJobGroup:
  • cancelStage: 根据 stageId 和 reason 取消 DatScheduler 对应的 Stage
  • setCheckpointDir: 配置 checkpoint 目录
  • defaultParallelism： 默认的并行度

伴生对象：

  • getOrCreate：
  • createTaskScheduler：Spark 功能的主要入口点。 返回 SchedulerBackend, TaskSchedueler，根据传入的 master 确定对应的 TaskSchedulerImpl 参数，确定 SchedulerBackend。
    • local[*], local[N], local-clusterxx spark://xxx, …

任务调度组件

任务调度相关组件：

  • DAGScheduler： class，负责高层的调度，将 Job 转换为 DAG 图
  • TaskScheduler：trait, 负责实际每个具体 Task 的物理调度执行，默认只有一种实现 TaskSchedulerImpl
    • 确定公共的逻辑，如决定作业执行的调度顺序，唤醒并启动推测任务
  • SchedulerBackend：trait, 有多种实现，分别对应不同的资源管理器, 完成应用程序向 Master 的注册

DAGScheduler

创建 Job, 将 DAG 中的 RDD 划分到不同的 Stage，提交 Stage 等

成员变量：

  • JobWaiter: 等待 DAGScheduler 的 Job 完成

主要的方法：

  • submitJob: 发送消息
  • handleJobSubmitted: 处理 Job 的提交
  • createResultStage： 创建 ResultStage
  • submitStage(stage)： 提交 ResultStage。根据 ResultStage 逐层调用父 stage, 从最上游 stage 开始逐步调用 TaskScheduler.submitTasks 方法提交 task 集合，最后提交 ResultStage 的 task 集合。
  • submitMissingTasks(stage,jobId): 提交 tasks, 当 stage 的父 stage 可用时调用。

TaskScheduler

TaskSchedulerImpl： TaskScheduler 唯一的实现，传入 SparkContext, maxTaskFailures, isLocal 作为构造器。为 TaskSet 创建和维护一个 TaskSetManager 并追踪任务的本地性和错误信息，会向 DAGScheduler 汇报执行情况。

  • 先调用 initialize / start，之后通过 submitTasks 方法提交 task sets
  • initialize： 确定调度模式，FIFO/FAIR，选择对应的 schedulableBuilder，根据参数构建资源池
  • start: SchedulerBackend 启动，启动 speculative 执行线程
  • submitTasks： 提交 TaskSet

TaskSetManager: 根据 taskSet、maxTaskFailures, boacklistTracker 进行初始化，运行任务执行失败，可根据黑名单进行剔除。

调度策略： 将 TaskSetManager 放到任务队列后的处理

  • FIFO: TaskSetManager 根据先后顺序进入队列，出队列时取先进入的，为默认策略，只有一个 TaskSetManager 池。
  • Fair 调度策略： 先对子 Pool 进行排序，再对子 Pool 中的 TaskSetManager 进行排序

SchedulerBackend

一个 trait, 调度系统的后端接口，允许在 TaskSchedulerImpl 下插入不同的系统。有多种实现，如 StandaloneSchedulerBackend、LocalSchedulerBackend、CoarseGrainedSchedulerBackend。

  • reviveOffers： 在 TaskSchedulerImpl 的 submitTasks(taskSet) 中调用

CoarseGrainedSchedulerBackend:

  • start: 对 driverEndpoint 进行注册
  • makeOffers： 将集群的资源以 Offer 的方式发给上层的 TaskScheduler 处理
