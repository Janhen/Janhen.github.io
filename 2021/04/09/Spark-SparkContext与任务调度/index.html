Spark-Master、Worker 启动和 SparkContext 初始化

Master 与 Worker 解析

RPC 框架

Spark 中不同组件之间的通信、jar 的上传、Shuffle 数据的传输、Block 数据的复制与备份都是基于 RPC 来实现的。

  • RpcEndpoint： 表示一个消息通信体，可以接收、发送、处理消息。
    • construct: 构造器
    • onStart：启动
    • receive*: 用来接收RpcEndpoint send 或 ask 过来的消 息
      • receive: 接收由RpcEndpointRef.send方法发送的消息，不需要立即处理
      • receiveAndReply: 接受消息并回复。接收由RpcEndpointRef.ask发送的消息，需要立即处理
    • onStop：资源回收
  • RpcEnv： 管理着整个 RpcEndpoint 的生命周期
    • 根据name或uri注册endpoints、管理各种消息的处理、停止endpoints。
  • RpcEndpointRef: 对远程 RpcEndpoint 的一个引用。当需要向一个具体的 RpcEndpoint 发送消息时，需要获取到该 RpcEndpoint 的引用，然后通过该引用发送消息。
    • send: Send-and-forget, 无需等待响应
    • ask: 等待通信对端响应，返回 Future 异步获取返回结果

Master 的运行

为 RpcEndpoint，实现了 ThreadSafeRpcEndpoint 接口，是线程安全的。

  • onStart：

    • 最主要进行恢复，Master 的 HA 方式：
      • ZOOKEEPER。
      • FILESYSTEM。基于文件系统的单点恢复。
      • CUSTOM。允许用户自定义 HA 的实现
      • NONE。未配置HA，不会持久化集群的数据

  • receive:

    • RegisterWorker： 注册 worker 的处理，接受到后，

      按照创建 workerInfo

      → 注册 workerInfo

      → 向 Worker 发送 RegisteredWorker 消息，表示注册完成

      → 调用 schedule 方法进行资源调度

    • RegisterApplication: 注册应用程序App, 如果是 Master 是 standby，则忽略这个消息; 否则注册 application；返回结果并且开始调度

    • Heartbeat： 心跳消息处理。记录 worker 节点的最后心跳时间，用于过期剔除 worker

    • MasterChangeAcknowledged： master 更改消息。将 appId 的 app 的状态置为 WAITING，为切换 Master 做准备。

    • CheckForWorkerTimeOut： 检查 worker 是否超时，移除超时没有发送心跳的 worker 节点

  • receiveAndReply:

    • RequestKillDriver: kill driver 的请求
    • RequestDriverStatus： driver 的状态请求
    • RequestMasterState： 请求 master 的状态
    • …

Worker 的运行

  • onStart
    • 向注册 Master
    • 启动额外的 shuffle 服务
    • work节点申请资源
    • 启动 work 节点 的监控(MetricsSystem)
  • receive:
    • SendHeartbeat: Worker 收到 SendHeartbeat 消息后，会向 Master 转发 Heartbeat 消息。
    • LaunchDriver: 启动 driver
    • …

交互

Worker 发送心跳

Master 处理 RegisterWorker 的消息

Master 记录心跳的情况

  • 接收注册的消息

  • 记录最后一次发送的心跳时间

  • 回复注册的消息，发送心跳

Worker 发送心跳

  • 每隔 60 / 4 秒发送一次

  • 自己给自己发送消息

额外的组件

  • 后台定时转发线程 forwordMessageScheduler

SparkContext

Spark Context 是程序的入口点。初始化时包含内部组件的创建和准备，涉及网络通信、分布式、消息、存储、计算、调度、缓存、度量、清理、文件服务和 UI 等方面。链接 Spark 集群，创建 RDD，累加器和广播变量。在 App 中将外部的数据转换为 RDD, 建立第一个 RDD，时 RDD 血缘关系的根，DAG 的根源。

内部组件

  • sparkConf： 设置 K-V 格式的参数，配置的优先级高于配置文件
  • sparkEnv： Spark 的运行环境, 所有的 Executor 都有对应的指定环境，开发者 API，无法对外调用，可以将数据存储在存储体系中、使用计算引擎对任务进行处理、可以在节点之间进行通信等。包含如下组件：
    • RpcEnv: 使用 Netty 实现组件之间的通信
    • ShuffleManager： 管理本地和远程的 Block 数据的 Shuffle 操作，根据 spark.shuffle.manager 进行指定，默认使用的是 SortShuffleManager。
    • BlockManager： 缓存持久化管理，为 Spark 的 Storage 模块，本身也是 Master / Slave 的
    • MemoryManager： 内存管理, 默认使用统一内存管理 UnifiedMemoryManager, 有过期的 StaticMemoryManager
    • MapOutPutTracker: 跟踪 Map 阶段任务的输出状态，方便 reduce 阶段任务获取地址和中间结果，一般是 MapStatus 的处理。
    • SerializerManager： 序列化管理器，可通过 spark.serializer 配置序列化的实现
    • serializer： 序列化器
    • BroadcaseManager： 广播变量管理，使用 BT 协议分发广播变量，避免 Driver 分发广播变量给所有的 Executor
    • MetricsSystem: 度量系统，有 source, sink，可将度量信息放到 SparkUI 上展示
    • SecurityManager：安全管理
  • ConsoleProgressBar: _progressBar，进度条显示
  • SparkUI: Web 页面展示，将度量信息展示出来，展示 RDD 的依赖关系，以 SparkListenerEvent 投递给 LiveListener，从 SparkListener 中读取数据展示
  • EventLoggingListener: 事件日志监听器
    • 配置 spark.eventLog.enabled=true 开启
  • ExecutorAllocationManager: Executor 动态分配管理器
    • 可动态的分配最小 executor 数量，动态分配最大 executor 数量，每个 executor 可运行的 tasks 数量等
  • ContextCleaner: 清理超出 app 范围的 rdd、shuffleDependency 和 broadcast 对象
  • DAGScheduler： DAG 调度器, 负责创建 job，将 DAG 的 RDD 划分为不同的 Stage，提交 Stage
  • TaskScheduler：任务调度器，按照调度算法对 ClusterManager 已经分配给应用程序的资源进行二次调度后分配任务，调度的 Task 是 DAGScheduler 创建的，DAGScheduler 是 TaskScheduler 的前置调度器。
  • SchedulerBackend： 用于对接不同的资源管理系统，允许在 TaskSchedulerImpl 下插入不同的系统，如 StandaloneSchedulerBackend、CoarseGrainedSchedulerBackend、LocalSchedulerBackend
  • LiveListenerBus： 事件总线，接受各个使用方的事件，异步方式对事件进行匹配和处理
  • SparkStatusTracker： 状态追踪器，提供对作业、stage 的监控信息

SparkContext 方法：

  • **runJob**: action 内部调用的
  • submitJob: 提交一个作业以供执行，并返回一个包含结果的 FutureJob。
  • setCheckpointDir: 配置 checkpoint 目录，用于容错
  • defaultParallelism： 返回默认的并行度
  • submitMapStage: 根据传入的 ShuffleDependency 提交 MapStage，为内部 API
  • cancelJobGroup:
  • cancelStage: 根据 stageId 和 reason 取消 DatScheduler 对应的 Stage
  • makeRDD / parallelize: 根据集合创建 RDD
  • textFile：根据文件创建 RDD
  • register： 注册累加器
  • longAccumulator/collectionAccumulator/doubleAccumulator： 返回默认的累加器
  • broadcast：广播变量

SparkContext 伴生对象：

  • getOrCreate：创建
  • **createTaskScheduler**：Spark 功能的主要入口点。 返回 SchedulerBackend, TaskSchedueler，根据传入的 master 确定对应的 TaskSchedulerImpl 参数，确定 SchedulerBackend。
    • local[*], local[N], local-clusterxx spark://xxx, …

初始化过程

  1. 初始设置

对 SparkConf 进行复制，校验各种配置信息，必须包含 spark.master 和 spark.app.name

  1. 创建 SparkEnv

  2. 创建 SparkUI

spark.ui.enabled=true 进行开启 ui

  1. Hadoop 相关配置，参数前缀为 spark.hadoop 的属性进行赋值，如配置 hdfs 的文件大小

  2. Executor 环境变量，如 executor 的执行内存和额外的 JVM 参数，默认执行内存为 1G，生产中需要加大

  3. 注册 HeartbeatReceiver 心跳接收器

生产中 driver 和 executor 可能运行在不同的节点上，driver 创建心跳接收器接受 executor 的心跳

  1. 创建 TaskScheduler、SchedulerBackend
  • TaskScheduler 进行任务的提交，负责发送任务到 Cluster 运行，失败重试
  1. 创建和启动 DAGScheduler，负责创建 Job, 将 DAG 中的 RDD 划分为 Stage 并提交 Stage

维护 jobId 与 stageId、Stage、ActiveJob、缓冲 rdd 的 partition 位置信息

  1. 启动 TaskScheduler、SchedulerBackend

  2. 启动测量系统 MetricsSystem

注册 sources → 注册 sinks → 将 sinks 增加 jetty 的 serveletContextHandler → 遍历 sinks 相关的 servletContextHandler，调用 attachHandler 将其绑定到 SparkUI 上

  1. 创建事件日志监听器，可将事件持久化到存储的监听器，为可选的组件

  2. 创建和启动 ExecutorAllocationManager，对已经分配的 Executor 管理，可选的，默认不创建

  3. ContextCleaner 的创建与启动： 清理超出应用范围的 RDD, ShuffleDependency, Broadcase 对象

  4. 自定义 SparkListener 与启动事件

  5. Spark 环境更新： 在 SpringContext 初始化时，对当前 ENV 有影响，更新新环境

将环境的 JVM 参数、Spark 属性、系统属性、classPath 作为环境明细信息

  1. 投递应用程序启动事件

向 listenerBus 发送 SparkListenerApplicationStart 事件

  1. 测量系统添加 Source：创建 DAGSchedulerSource, BlockManagerSource, ExecutorAllocationManagerSource

  2. 将 SparkContext 标记为激活： 从 contextBeingConstructred → activeContext

相关配置

  • spark.executor.heartbeatInterval： executor 发送心跳的间隔, 默认 10秒
